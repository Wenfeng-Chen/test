## 问题一，实现分布式 RESTful API接口/性能测试框架，80 分

API接口的功能/性能测试是后端服务快速迭代过程中保证质量和一致性的重要环节.
我们要实现一个简化版的接口与性能测试框架, 支持通过编写测试脚本(python/js等) 实现对 RESTful API 的简单测试.

要求：
1. 支持 HTTP 协议所有的请求方法, 参考 https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Methods
2. 支持单机模式, 向被测服务发送请求
3. 支持分布式模式，协调局域网内多台设备同时向被测服务发起请求
4. 测试结果需持久化存储（不限制数据库类型，如 SQLite、PostgreSQL、InfluxDB 均可）
5. 提供项目设计文档（架构图、主从通信协议、任务调度策略、数据聚合算法）

可选的模块：
1. 统计并聚合关键性能指标：QPS / TPS、平均响应时间、最小/最大响应时间、TP90、TP95、TP99、错误率
2. 实现一个 CLI 工具，通过命令行提交压测任务、查看运行状态或导出报告
3. 实现一个 Web 前端页面，实时展示压测过程的吞吐量曲线和响应时间分布图
4. 编写 Dockerfile / docker-compose.yml，方便一键部署 Master 节点及依赖数据库
5. 编写单元测试，覆盖任务分发、数据聚合计算等核心逻辑
6. 支持多种压测模式：
   - 固定并发数模式（如始终保持 100 个并发）
   - RPS 模式（每秒固定请求数）
   - 阶梯加压模式（按时间逐步增加并发数）



## 问题二：自定义 LLM 评估与基准测试框架（90 分）

当前大模型评测（如 MMLU、GSM8K）通常需要复杂的脚本流程。你的任务是实现一个可扩展的评估框架，允许用户自定义测试集和评分逻辑。

要求：
1. 提供 CLI，支持运行指定的测试套件（例如：`evaluate --suite math --model gpt-4`）
2. evaluator 模块：支持自定义评估指标（如准确率、BLEU、ROUGE，或基于正则的规则评分）
3. tools 模块：提供数据集加载、模型 API 调用重试、结果缓存等工具函数
4. 支持对接至少一种大模型 API（OpenAI / Anthropic / 本地 vLLM），并将模型的原始输出与预期答案进行比对
5. 提供项目设计文档（模块划分、评估流程、指标扩展接口）

可选模块：
1. 支持 `vibe coding` 模式：当模型回答格式错误时，自动重试并修正格式
2. providers 模块：统一管理多模型服务商，支持结果对比（A/B 测试）
3. skills 模块：允许用户通过 YAML 配置“评估技能包”，包括提示词模板和断言逻辑
4. 生成可视化评估报告（HTML/PDF/Markdown 格式），包含混淆矩阵或评分雷达图


## 问题三：实现面向大文件的分布式去重存储引擎（100 分）

类似 Git 对大文件的低效存储，云盘（如 Dropbox、百度网盘）在上传大文件时，如果用户只修改了文件的一小部分，服务端通常采用可变分块（Content-Defined Chunking, CDC）算法（如基于 Rabin 指纹）来切割文件，以更精准地识别重复数据块，从而极大节省带宽和存储空间。

要求：
1. 实现一个去重存储后端，基于可变分块算法（而非固定 64KB）切割文件：
   - 利用滚动哈希（如 Rabin-Karp）根据文件内容确定分块边界（即内容变化时，分界点偏移）
   - 计算每个块的哈希（SHA-256），并以内容寻址（Content-Addressable）方式存储
   - 存储两个仅修改几字节的大文件版本时，仅新文件变化处之后的少量块发生变动，极大节省空间
2. 提供 API 或 CLI：
   - `store <file-path>`：存储文件并返回根哈希（类似 Git 的 Tree Hash）
   - `fetch <root-hash> <output-path>`：根据根哈希还原完整文件
   - `stats <file-path>`：显示该文件被切分成了多少块以及全局去重率
3. 元数据（块索引、文件哈希映射）持久化到本地数据库（如 RocksDB、LevelDB 或 SQLite）
4. 编写测试脚本，验证存储效率：生成 1GB 文件，修改 100 字节后存第二个版本，对比您的引擎与全量复制所占用的磁盘空间差异
5. 提供项目设计文档（CDC 分块策略、哈希表结构、数据恢复流程）

可选模块：
1. 实现垃圾回收（GC），清理未被任何文件引用的孤立数据块
2. 支持将块数据存储到远端对象存储（如 OSS、MinIO、RustFS），实现云原生去重
3. 实现块缓存（LRU）机制，加速频繁读取的大文件头部块
4. 提供 Prometheus 监控指标（存储量、分块耗时、上传吞吐量）
